Временная сложность бинарного поиска в худшем случае составляет O(log N), где N — количество элементов в списке. 

Объяснение: Бинарный поиск работает по принципу "разделяй и властвуй". На каждом шаге он делит пространство поиска (список) пополам. Таким образом, количество итераций, необходимых для нахождения элемента (или определения его отсутствия), — это количество раз, которое можно поделить N на 2, пока не останется один элемент, что математически выражается как логарифм N по основанию 2 (log₂N). Это делает его значительно быстрее, чем линейный поиск O(N), особенно для больших списков. 

Пространственная сложность бинарного поиска составляет O(1) (для итеративной реализации). 

Объяснение: Алгоритм использует лишь небольшое постоянное количество дополнительной памяти для хранения переменных (таких как left, right, mid), независимо от размера входного списка N.
